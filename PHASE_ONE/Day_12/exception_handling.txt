Exception Handling:
Exception Handling prevents crashes by catching errors
and handling them gracefully — like a safety net.

Real world analogy:

Think of a pilot. When an engine fails, the plane doesn't
just drop from the sky. The pilot has backup systems
and emergency procedures.

Exception handling = your program's emergency procedure.


The Big Picture:
WITHOUT Exception Handling    WITH Exception Handling
──────────────────────────    ───────────────────────
Error → Program CRASHES   →   Error → Caught → Handle it
User sees ugly traceback  →   User sees friendly message
Program stops completely  →   Program keeps running

Types of Common Exceptions: 

Exception           When it happens

ValueError              Wrong value type: int("hello")
TypeError               Wrong type operation: "Ali" + 5
ZeroDivisionError       Dividing by zero: 10 / 0
FileNotFoundError       File doesn't exist
IndexError              List index out of range
KeyError                Dictionary key doesn't exist
NameError               Variable not defined
AttributeError          Method doesn't exist on object
